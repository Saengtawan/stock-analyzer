# 🎯 Portfolio Monitor - เงื่อนไขการออกจาก Portfolio

## 📋 สรุป: หุ้นจะหายจาก Portfolio เมื่อไหร่?

หุ้นจะถูกเอาออกจาก Portfolio Monitor เมื่อ**ถูกเงื่อนไข Exit Rules อย่างใดอย่างหนึ่ง**

ระบบใช้ **11 Exit Rules** แบ่งเป็น 4 ระดับความสำคัญ:

---

## 🔴 CRITICAL (ต้องออกทันที)

### 1. ✅ TARGET_HIT - ถึงเป้ากำไร
**เงื่อนไข:**
- กำไร ≥ **+4.0%** ✅

**ตัวอย่าง:**
```
ซื้อที่ $100
ขายที่ $104+ (กำไร 4%+)
→ ออกจาก portfolio ทันที
```

**เหตุผล:** ทำกำไรได้ตามเป้าแล้ว ออกเพื่อลดความเสี่ยง

---

### 2. ❌ HARD_STOP - ขาดทุนเกินกำหนด
**เงื่อนไข:**
- ขาดทุน ≤ **-3.5%** ❌

**ตัวอย่าง:**
```
ซื้อที่ $100
ราคาตก $96.50 (ขาดทุน -3.5%)
→ ตัดขาดทุนทันที
```

**เหตุผล:** ป้องกันขาดทุนมากขึ้น (เดิมคือ -6% แต่ปรับเป็น -3.5% เพื่อลดความเสียหาย)

---

### 3. 📉 TRAILING_STOP - ตกจากจุดสูงสุด
**เงื่อนไข:**
- เคยขึ้นเกินราคาซื้อแล้ว (มีกำไร)
- แล้วตกจากจุดสูงสุด ≥ **-3.5%**

**ตัวอย่าง:**
```
ซื้อที่ $100
ขึ้นไป $110 (กำไร +10%)
ตกกลับมา $106.15 (ตกจาก peak -3.5%)
→ ออกทันทีเพื่อรักษากำไร
```

**เหตุผล:** ล็อคกำไร ไม่ให้กำไรที่มีหายไป

---

## 🟠 HIGH Priority (สัญญาณแรง)

### 4. 🔻 SMART_GAP_DOWN - เปิดตลาด Gap Down แรง
**เงื่อนไข:**
- ราคาเปิด < ราคาปิดเมื่อวาน ≥ **1.5%**
- และขาดทุนรวม > **1.0%**
- ถือมาแล้ว ≥ 1 วัน

**ตัวอย่าง:**
```
เมื่อวานปิด $100, วันนี้เปิด $98.50 (Gap down -1.5%)
ขาดทุนรวม -1.2%
→ ออกทันที (สัญญาณลงแรง)
```

**เหตุผล:** Gap down แรง + ขาดทุนอยู่แล้ว = สัญญาณไม่ดี

---

### 5. 📉 SMART_BREAKING_DOWN - ราคาตกแรงในวัน
**เงื่อนไข:**
- ราคาตกภายในวัน > **2.0%**
- และขาดทุนรวม > **0.5%**
- ถือมาแล้ว ≥ 2 วัน

**ตัวอย่าง:**
```
เช้า $100 → เย็น $98 (ตก -2%)
ขาดทุนรวม -0.8%
→ ออกทันที (breakdown)
```

**เหตุผล:** ตกแรงในวัน + ขาดทุนอยู่แล้ว = อาจตกต่อ

---

## 🟡 MEDIUM Priority (สัญญาณปานกลาง)

### 6. 📊 SMART_VOLUME_COLLAPSE - Volume หาย
**เงื่อนไข:**
- Volume วันนี้ < **50%** ของ avg 10 วัน
- และขาดทุนรวม > **1.0%**
- ถือมาแล้ว ≥ 2 วัน

**ตัวอย่าง:**
```
Volume เฉลี่ย 10M shares
วันนี้ volume แค่ 4M (40% ของเฉลี่ย)
ขาดทุน -1.2%
→ ออก (ไม่มีคนสนใจซื้อ)
```

**เหตุผล:** Volume ต่ำ + ขาดทุน = ไม่มีแรงซื้อ อาจตกต่อ

---

### 7. 🎢 SMART_FAILED_PUMP - ปั๊มแล้วตก
**เงื่อนไข:**
- เคยขึ้นไป > **3%** แล้ว
- แต่ตอนนี้ตกกลับต่ำกว่าราคาซื้อแล้ว
- ถือมาแล้ว ≥ 2 วัน

**ตัวอย่าง:**
```
ซื้อที่ $100
ขึ้นไป $104 (กำไร +4%)
ตกกลับมา $99.50 (ขาดทุน -0.5%)
→ ออก (pump & dump pattern)
```

**เหตุผล:** ขึ้นแล้วตกกลับ = อาจเป็น pump & dump หรือ fake rally

---

### 8. 📉 SMART_SMA20_BREAK - ทะลุ SMA20 ลง
**เงื่อนไข:**
- ราคา < SMA20 เกิน **1%**
- และขาดทุนอยู่
- มีข้อมูลอย่างน้อย 20 วัน
- ถือมาแล้ว ≥ 2 วัน

**ตัวอย่าง:**
```
SMA20 = $100
ราคาปัจจุบัน = $99 (ต่ำกว่า SMA20 -1%)
ขาดทุน -0.8%
→ ออก (ทะลุ support)
```

**เหตุผล:** ทะลุ SMA20 = แนวรับเสีย อาจตกต่อ

---

## 🟢 LOW Priority (สัญญาณอ่อน)

### 9. 😔 SMART_WEAK_RSI - RSI อ่อนแรง
**เงื่อนไข:**
- RSI < **35** (oversold แต่ไม่มีแรงฟื้น)
- และขาดทุน > **2.0%**
- มีข้อมูลอย่างน้อย 15 วัน
- ถือมาแล้ว ≥ 2 วัน

**ตัวอย่าง:**
```
RSI = 32 (oversold)
ขาดทุน -2.5%
→ ออก (ตก oversold แล้วไม่ฟื้น)
```

**เหตุผล:** RSI ต่ำ + ขาดทุนมาก = อ่อนแรงมาก อาจตกต่อ

---

### 10. 🔄 SMART_MOMENTUM_REVERSAL - Momentum กลับตัว
**เงื่อนไข:**
- วันแรก: ขึ้น > **1%**
- วันที่สอง: ตก > **1.5%**
- กำไรรวม < **1%**
- ถือมาแล้ว ≥ 2 วัน

**ตัวอย่าง:**
```
Day 1: +2%
Day 2: -2.5%
กำไรรวม: -0.5%
→ ออก (momentum reversal)
```

**เหตุผล:** ขึ้นแล้วกลับตก = momentum หมด

---

### 11. ⏰ MAX_HOLD - ถือนานเกินไป
**เงื่อนไข:**
- ถือมาแล้ว ≥ **30 วัน**

**ตัวอย่าง:**
```
ซื้อวันที่ 1 ธ.ค.
วันที่ 31 ธ.ค. (ครบ 30 วัน)
→ ออกอัตโนมัติ (ไม่ว่ากำไรหรือขาดทุน)
```

**เหตุผล:** Swing trading ไม่ควรถือเกิน 30 วัน (เงินติดเยอะ)

---

## 📊 Priority System

Rules จะถูกตรวจสอบตามลำดับความสำคัญ:

```
1. CRITICAL Rules ก่อน (TARGET_HIT, HARD_STOP, TRAILING_STOP)
   → ต้องออกทันที ไม่ต้องเช็ค rule อื่น

2. HIGH Priority (SMART_GAP_DOWN, SMART_BREAKING_DOWN)
   → สัญญาณแรง ออกเพื่อป้องกันขาดทุนเพิ่ม

3. MEDIUM Priority (SMART_VOLUME_COLLAPSE, SMART_FAILED_PUMP, SMART_SMA20_BREAK)
   → สัญญาณปานกลาง ออกเมื่อมีหลายสัญญาณรวมกัน

4. LOW Priority (SMART_WEAK_RSI, SMART_MOMENTUM_REVERSAL, MAX_HOLD)
   → สัญญาณอ่อน หรือ timeout
```

**Rule แรกที่เข้าเงื่อนไข จะทำให้ออกทันที** (ไม่เช็ค rule อื่นต่อ)


## 💡 คำถามที่พบบ่อย

### Q: หุ้นจะหายเมื่อไหร่?
**A:** เมื่อ**ถูกเงื่อนไขใดเงื่อนไขหนึ่ง**จาก 11 rules ข้างบน

---

### Q: ต้องถึง -3.5% ถึงจะออกหรือเปล่า?
**A:** ไม่จำเป็น! มี **SMART exits** ที่ออกก่อนถึง -3.5%

ตัวอย่าง:
- Gap down -1.5% + ขาดทุน -1% → ออกที่ขาดทุน ~-2.5% (ก่อนถึง -3.5%)
- Breaking down -2% + ขาดทุน -0.5% → ออกที่ขาดทุน ~-2.5%

**ค่าเฉลี่ยจาก backtest: ขาดทุน -2.98%** (ไม่ถึง -3.5%!)

---

### Q: ทำไมไม่เอา MSFT ออกทั้งที่ขาดทุน -2%?
**A:** เพราะยัง**ไม่ถูกเงื่อนไขใดๆ**:
- ยังไม่ถึง -3.5% (HARD_STOP)
- ไม่มี gap down ≥ -1.5%
- ไม่มี breaking down ≥ -2% ในวัน
- etc.

**หุ้นจะอยู่จนกว่าจะถูกเงื่อนไข** (อาจฟื้นกลับมา หรือ ตกต่อจนถูก HARD_STOP)

---

### Q: กำไร +3.5% ทำไมไม่ออก?
**A:** เพราะ target คือ **+4.0%** ไม่ใช่ 3.5%

ถ้าต้องการออกเร็วขึ้น ปรับ threshold:
```python
exit_rules.update_threshold('TARGET_HIT', 'target_pct', 3.5)
```

---

### Q: ถือครบ 30 วันแล้ว ขาดทุน -1% จะออกไหม?
**A:** **ออก!** ตาม MAX_HOLD rule (ถือเกิน 30 วัน)

แม้จะยังไม่ถึงเป้ากำไร หรือยังขาดทุนไม่ถึง stop loss

---

### Q: สามารถปิด rule บางตัวได้ไหม?
**A:** ได้!

```python
# ปิด MAX_HOLD (ถ้าอยากถือนานกว่า 30 วัน)
exit_rules.disable_rule('MAX_HOLD')

# ปิด SMART_WEAK_RSI (ถ้าไม่อยากใช้ RSI)
exit_rules.disable_rule('SMART_WEAK_RSI')

# เปิดคืน
exit_rules.enable_rule('MAX_HOLD')
```

---

### Q: ปรับ threshold ได้ไหม?
**A:** ได้!

```python
# เปลี่ยน target จาก 4% เป็น 5%
exit_rules.update_threshold('TARGET_HIT', 'target_pct', 5.0)

# เปลี่ยน hard stop จาก -3.5% เป็น -5%
exit_rules.update_threshold('HARD_STOP', 'stop_pct', -5.0)

# เปลี่ยน trailing stop จาก -3.5% เป็น -5%
exit_rules.update_threshold('TRAILING_STOP', 'drawdown_pct', -5.0)
```
