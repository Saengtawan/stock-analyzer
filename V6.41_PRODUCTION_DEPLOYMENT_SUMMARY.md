## 🚨 CRITICAL - Must Read Before Deployment

This release fixes **10 critical race conditions** discovered through adversarial testing. These bugs could cause:
- Duplicate positions (lost tracking)
- Ghost positions (positions exist but not tracked)
- Memory leaks (UI)
- Trading freeze (deadlock)
- Data loss (DB sync failures)

**All bugs are FIXED and tested.** Deploy immediately to production.

## 🔐 Lock Architecture (v6.41)

| Lock | Purpose | Coverage | Notes |
|------|---------|----------|-------|
| `_positions_lock` | Position dict | ✅ Re-checks symbol | Prevents double-buy |
| `_queue_lock` | Signal queue | ✅ Thread-safe | Already covered (Phase 4) |
| `_scan_lock` | Scan mutex | ✅ Has watchdog | 5-min timeout |
| `_opening_window_lock` | **NEW** | ✅ Counter protection | Atomic read/write |
| `_close_locks` | Per-symbol close | ✅ Already safe | Dict of locks |

**Critical Sections Protected:**
1. Position creation: Re-check symbol inside `_positions_lock`
2. Queue execution: Check memory inside `_queue_lock`
3. DB sync: Fail-fast + rollback on error
4. Opening counters: All operations inside `_opening_window_lock`
5. Scan acquisition: Watchdog checks before blocking

## 🚀 Deployment Steps

### **Pre-Deployment**

1. **Backup Current State:**
   ```bash
   # Backup databases
   cp data/trade_history.db data/trade_history.db.backup_$(date +%Y%m%d_%H%M%S)
   cp data/rapid_trader.db data/rapid_trader.db.backup_$(date +%Y%m%d_%H%M%S)

   # Backup logs
   tar -czf logs_backup_$(date +%Y%m%d_%H%M%S).tar.gz logs/
   ```

2. **Pull Latest Code:**
   ```bash
   git pull origin master
   # Should be at commit: 0908925 (or later)
   ```

3. **Verify Commits:**
   ```bash
   git log --oneline -5
   # Should show:
   # 0908925 Fix MEDIUM priority bugs - Scan deadlock + Opening window race
   # f2c0128 Fix CRITICAL backend race conditions
   # 7eaf72a Fix critical socket lifecycle bugs
   # 99325e5 Fix Phase 1 UI race conditions
   ```

### **Deployment**

4. **Stop Services:**
   ```bash
   # If using systemd
   sudo systemctl stop auto-trading

   # Or manually
   pkill -f auto_trading_engine.py
   ```

5. **Run Tests (Optional but Recommended):**
   ```bash
   python3 tests/test_race_conditions_v6_41.py
   # All tests should pass
   ```

6. **Start Services:**
   ```bash
   # If using systemd
   sudo systemctl start auto-trading

   # Or manually
   nohup python3 src/auto_trading_engine.py > nohup.out 2>&1 &
   ```

7. **Verify Startup:**
   ```bash
   tail -f logs/auto_trading.log
   # Should see: "Engine started successfully" (no errors)
   ```

### **Post-Deployment**

8. **Monitor for 1 Hour:**
   ```bash
   # Terminal 1: Watch logs for race conditions
   ./scripts/monitor_race_conditions.sh 60

   # Terminal 2: General log monitoring
   tail -f logs/auto_trading.log
   ```

9. **Check UI:**
   - Open browser: http://localhost:5000
   - Verify no flickering in header/positions
   - Check Network tab: No duplicate API calls when socket connected
   - Test disconnect: Turn off WiFi → verify polling starts → reconnect → verify polling stops

10. **24-Hour Monitoring:**
    - Check logs daily for race condition warnings
    - Monitor position reconciliation (should be clean)
    - Verify no RuntimeError in logs
    - Confirm scan lock never stuck > 5 min

## 🚨 Rollback Plan

If critical issues appear:

1. **Stop Trading Immediately:**
   ```bash
   sudo systemctl stop auto-trading
   # Or send emergency stop via UI
   ```

2. **Revert to Previous Version:**
   ```bash
   git checkout <previous_commit_hash>
   # Before v6.41: likely v6.37 or earlier
   ```

3. **Restore Databases (if corrupted):**
   ```bash
   cp data/trade_history.db.backup_* data/trade_history.db
   cp data/rapid_trader.db.backup_* data/rapid_trader.db
   ```

4. **Restart:**
   ```bash
   sudo systemctl start auto-trading
   ```

5. **Report Issue:**
   - Document exact error in logs
   - Note which bug fix failed
   - Create GitHub issue with full trace
